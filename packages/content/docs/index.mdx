Domain UI is a **component registry** that provides specialized input components with built-in validation for domain-specific use cases. Built with modern React patterns using Radix UI primitives and Tailwind CSS, it offers regex-based validation, country-specific inputs, and development tools.

This is **not a traditional component library**. It's a registry of components that you copy into your project and own completely.

> "Domain-specific components that handle validation, formatting, and compliance out of the box."

## What Makes Domain UI Different

### Regex-Powered Validation

At the core of Domain UI is a powerful regex validation system that provides:

- **Simple Validation**: Basic regex matching with instant feedback
- **Incremental Validation**: Advanced NFA-based validation that guides users as they type
- **Real-time Feedback**: Visual indicators for valid/invalid states
- **Uppercase Transformation**: Automatic formatting for components that require it

### Country-Specific Components

Domain UI provides ready-to-use components for different countries' requirements:

- **India**: PAN input (`ABCDE1234F`) and Passport input (`A1234567`) with optional labels
- **USA**: Passport input validation (supports multiple formats: `A12345678`, `123456789`, `A1234567`)
- **UK**: Passport input (`ABC123456`) with built-in label
- **Global**: Extensible regex patterns for any country or document type

### Development Tools

Built-in tools to enhance your development workflow:

- **Responsive Preview**: Test components across 13 breakpoints (xxs to 7xl) with resizable panels
- **Registry Display**: Showcase component code with syntax highlighting and file tree navigation
- **NFA Engine**: Advanced Non-deterministic Finite Automaton for incremental regex validation
- **Custom Hooks**: Reusable validation logic with `use-simple-regex` and `use-incremental-regex`

## Core Components

### RegexInput

The foundation component that powers all domain-specific inputs:

```tsx
import { RegexInput } from "@/components/domain-ui/regex-input"

// Basic usage
<RegexInput 
  regex={/^[A-Z]{3}[0-9]{4}$/}
  placeholder="Enter code"
  transformToUppercase
/>
```

### PanInput (India)

Pre-configured for Indian PAN card validation:

```tsx
import { PanInput } from "@/components/domain-ui/india/pan-input"

<PanInput 
  placeholder="Enter PAN number"
  onValidation={(status) => console.log(status)}
/>
```

### Passport Inputs

Country-specific passport validation:

```tsx
// USA Passport (multiple formats supported)
import { PassportInput as UsaPassportInput } from "@/components/domain-ui/usa/passport-input"

// UK Passport (with built-in label)
import { PassportInput as UkPassportInput } from "@/components/domain-ui/uk/passport-input"

// Indian Passport (configurable label)
import { PassportInput as IndianPassportInput } from "@/components/domain-ui/india/passport-input"

<IndianPassportInput showLabel={false} /> // Without label
<UkPassportInput /> // Always includes "UK Passport" label
```

## Installation

Install components directly from the registry:

```bash
# Install a specific component
pnpm dlx shadcn@latest add "http://localhost:3000/r/pan-input.json"

# Install with incremental validation
pnpm dlx shadcn@latest add "http://localhost:3000/r/pan-input-incremental.json"

# Install development tools
pnpm dlx shadcn@latest add "http://localhost:3000/r/responsive-preview.json"
```

## Validation Types

### Simple Validation

Basic regex matching that validates the entire input:

- **Immediate feedback** when input matches or fails
- **Boolean validation** (valid/invalid)
- **Lightweight** and fast

### Incremental Validation

Advanced validation using NFA (Non-deterministic Finite Automaton) with Thompson's construction:

- **Character-by-character guidance** as users type
- **Partial validation** shows progress toward complete match
- **Smart rejection** of characters that make matching impossible
- **Complex regex support** including alternation patterns and quantifiers
- **Paste detection** with full validation on paste events
- **Better user experience** for complex patterns like passport formats

- **Testing tools**: Responsive preview and validation testing
