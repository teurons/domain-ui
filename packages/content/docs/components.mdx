## Core Components

### RegexInput

Base validation component using native `RegExp.test()` for simple pattern matching.

```tsx
import { RegexInput } from "@/components/domain-ui/regex-input"

<RegexInput 
  regex={/^[A-Z]{3}[0-9]{4}$/}
  placeholder="Enter code"
  transformToUppercase
  onValidation={(status) => console.log(status)} // "valid" | "invalid"
/>
```

**Props:**
- `regex: RegExp` - Pattern for validation
- `transformToUppercase?: boolean` - Auto-convert to uppercase
- `onValidation?: (status: ValidationStatusType) => void` - Validation callback
- Extends all native `input` props

### RegexInputIncremental

NFA-based incremental validation that guides users character-by-character.

```tsx
import { RegexInputIncremental } from "@/components/domain-ui/regex-input-incremental"

<RegexInputIncremental 
  regex={/^[A-Z]{5}[0-9]{4}[A-Z]{1}$/}
  placeholder="Enter PAN number"
  onValidation={(status) => console.log(status)} // "valid" | "incomplete" | "invalid"
/>
```

**Features:**
- Blocks invalid characters during typing
- Allows paste operations with full validation
- Three validation states: `valid`, `incomplete`, `invalid`
- Uses NFA algorithm for complex regex patterns with alternation

## Domain-Specific Components

### PanInput

Indian PAN (Permanent Account Number) validation with automatic uppercase conversion.

```tsx
import { PanInput } from "@/components/domain-ui/india/pan-input"

<PanInput 
  placeholder="Enter PAN number"
  onValidation={(status) => console.log(status)}
/>
```

**Pattern:** `^[A-Z]{3}[PCFTABGHLJE]{1}[A-Z]{1}[0-9]{4}[A-Z]{1}$`  
**Example:** `ABCDE1234F`

### PassportInput

Country-specific passport validation with multiple format support.

```tsx
// India - Optional label display
import { PassportInput } from "@/components/domain-ui/india/passport-input"
<PassportInput showLabel={false} /> // Pattern: ^[A-Z][0-9]{7}$

// USA - Multiple legacy formats
import { PassportInput } from "@/components/domain-ui/usa/passport-input"
<PassportInput /> // Patterns: ^[A-Z][0-9]{8}$, ^[0-9]{9}$, ^[A-Z][0-9]{7}$

// UK - Built-in country label
import { PassportInput } from "@/components/domain-ui/uk/passport-input"
<PassportInput /> // Pattern: ^[A-Z0-9]{9}$
```

## Development Tools

### ResponsivePreview

Resizable preview container with breakpoint indicators and dark mode support.

```tsx
import { 
  PreviewWrapper, 
  ChildPreview, 
  IFramePreview 
} from "@/components/domain-ui/responsive-preview"

// Direct child rendering
<ChildPreview config={{ showToolbar: true, darkMode: false }}>
  <YourComponent />
</ChildPreview>

// IFrame rendering
<IFramePreview 
  srcUrl="http://localhost:3000/preview" 
  height={600}
  config={{ showScale: true, showLabels: true }}
/>
```

**Features:**
- 13 breakpoints from `xxs` (80px) to `7xl` (1280px)
- Resizable panels with drag handles
- Configurable toolbar, scale bar, and labels
- Dark/light mode toggle

### RegistryItemDisplay

Code viewer with file tree navigation and syntax highlighting.

```tsx
import { RegistryItemDisplay } from "@/components/domain-ui/registry-item-display"

<RegistryItemDisplay url="http://localhost:3000/r/pan-input.json" />
```

**Features:**
- Fetches registry items via URL
- Collapsible file tree with directory expansion
- Syntax highlighting for TypeScript, TSX, JSON
- Single-file and multi-file layout support

## Technical Architecture

### Validation Hooks

**useSimpleRegex**: Basic regex validation using native `RegExp.test()`
- Returns: `{ value, onChange, isValid, validationStatus }`
- States: `"valid"` | `"invalid"`

**useIncrementalRegex**: NFA-based character-by-character validation
- Returns: `{ value, onChange, onPaste, isValid, validationStatus }`
- States: `"valid"` | `"incomplete"` | `"invalid"`
- Blocks invalid characters during typing
- Allows paste operations with full content validation

### Component Composition

All domain-specific components extend the core `RegexInput` or `RegexInputIncremental` components:

```tsx
// PanInput implementation
export function PanInput(props: PanInputProps) {
  return <RegexInput regex={PAN_REGEX} transformToUppercase {...props} />;
}
```
